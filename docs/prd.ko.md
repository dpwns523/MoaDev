# 제품 요구사항 문서

## 제품

`MoaDev`는 유용한 외부 콘텐츠를 수집한 뒤, 각 글을 구조화된 한국어 학습 자료로 변환해 제공하는 인증 기반 AI 지식 웹 애플리케이션이다.

첫 번째 릴리즈는 `Keek news`를 초기 기술 콘텐츠 소스로 사용하고, 이후 IT 기업의 기술 블로그로 확장한다. 장기적으로는 금융 지식까지 포함하는 더 넓은 지식 플랫폼으로 갈 수 있지만, 초기 제품은 기술 콘텐츠에 집중한다.

## 제품 가설

유용한 지식은 여러 사이트에 흩어져 있고, 문체도 제각각이며, 한국어로 빠르게 이해하기도 어렵다. `MoaDev`는 이 마찰을 세 가지 방식으로 줄여야 한다.

- 승인된 외부 소스에서 가치 있는 콘텐츠를 수집한다
- 원문 글을 구조화된 한국어 지식 산출물로 변환한다
- 사용자가 여러 플랫폼을 오가지 않고 한곳에서 지식에 접근하게 한다

## 대상 사용자

- 여러 사이트를 직접 확인하지 않고도 유용한 기술 콘텐츠를 추적하고 싶은 개발자
- 중요한 글을 한국어로 빠르게 이해하고 싶은 엔지니어와 기술 리드
- 단순 번역이 아니라 용어 설명과 개념 설명까지 함께 보고 싶은 학습자

## 핵심 문제

기술 지식은 뉴스 사이트와 기업 엔지니어링 블로그에 분산되어 있다. 사용자가 올바른 글을 찾더라도, 직접 번역하고 요약하고 낯선 용어를 해석하고 관련 개념과 연결하며 중요한 내용을 정리하는 데 시간이 든다.

이 문제는 세 가지 마찰로 나뉜다.

- 발견의 마찰: 유용한 콘텐츠가 여러 플랫폼에 흩어져 있다
- 이해의 마찰: 사용자가 직접 번역하고 해석해야 한다
- 정리의 마찰: 소스가 달라도 일관된 카테고리 기반 지식 뷰를 얻기 어렵다

## 목표

- 승인된 외부 기술 콘텐츠를 하나의 인증된 제품 경험으로 묶는다
- 단순 링크 피드가 아니라 구조화된 한국어 지식 형태로 각 글을 제공한다
- 번역, 요약, 용어 설명, 개념 설명을 결합해 사용자가 읽기에서 이해로 이동하게 한다
- 소스와 카테고리 기준으로 콘텐츠를 정리해 사용자가 노이즈가 아닌 의도를 갖고 탐색하게 한다
- 현재 모노레포와 서비스 경계 안에서 첫 릴리즈를 현실적으로 출시할 수 있게 한다

## 첫 릴리즈 범위

첫 릴리즈는 다음 화면과 기능을 가진 인증형 웹 애플리케이션이다.

- 콘텐츠에 접근하기 전 로그인 게이트
- 소스와 카테고리 기반 기사 목록 화면
- 구조화된 AI 산출물을 보여주는 기사 상세 화면
- 이미 처리된 콘텐츠를 다시 탐색하는 최소한의 지식 라이브러리 화면

첫 릴리즈는 다음 조건으로 시작한다.

- 초기 소스는 `Keek news`
- 소수의 기술 카테고리만 지원
- 승인된 소스에 대해서만 기사 수집과 AI 가공 수행

첫 릴리즈는 가능한 모든 소스를 수집하거나 모든 지식 분야를 지원하지 않는다.

## 주요 사용자 여정

1. 사용자가 로그인한다.
2. 사용자는 카테고리 기반 지식 홈으로 이동한다.
3. 사용자는 승인된 소스의 기사 하나를 선택한다.
4. 사용자는 다음 정보를 읽는다.
   - 기사 메타데이터
   - 소스 맥락
   - 줄 단위 한국어 번역
   - 간결한 요약
   - 용어 설명
   - 개념 설명
   - 관련 개념
5. 사용자는 카테고리와 소스 내비게이션을 이용해 관련 콘텐츠를 계속 탐색한다.

## MVP 기능

- 전체 콘텐츠에 대한 인증 기반 접근 제어
- 승인된 외부 소스 기준의 기사 수집
- 처리된 기사에 대한 카테고리 기반 탐색
- 구조화된 AI 지식 산출물을 보여주는 기사 상세 페이지
- 소스, 발행 시각, 카테고리, 처리 상태를 포함한 일관된 기사 메타데이터
- 기사 목록과 상세 화면을 렌더링할 수 있는 구조화된 API 계약

## 구조화된 기사 산출물 계약

처리된 각 기사는 최소한 아래 정보를 제공해야 한다.

- 기사 제목
- 소스 이름과 소스 URL
- 발행 시각과 수집 시각
- 카테고리와 태그
- 소스 정책이 허용하는 원문 일부 또는 정규화된 본문 세그먼트
- 정규화 세그먼트에 매핑되는 줄 단위 한국어 번역
- 짧은 요약
- 한국어 설명이 포함된 핵심 용어 목록
- 기사에서 등장한 중요 아이디어의 개념 설명
- 이어서 학습할 수 있는 관련 개념 또는 인접 주제
- 산출물이 불완전하거나 불확실할 때의 처리 상태와 품질 메모

## 콘텐츠 범위

초기 콘텐츠 범위:

- `Keek news`

근시일 내 확장 범위:

- IT 기업의 기술 블로그

보류 범위:

- 금융 지식 콘텐츠
- 비기술 일반 매체 기사
- 소스 승인 없이 사용자가 임의로 제출하는 URL

## 카테고리 모델

MVP 카테고리 모델은 작고 이해하기 쉬워야 한다. 첫 릴리즈는 다음과 같은 기술 주제를 중심으로 그룹화하는 것을 기본으로 한다.

- AI 및 머신러닝
- 백엔드 및 인프라
- 프론트엔드 및 웹 플랫폼
- 클라우드 및 DevOps
- 데이터 및 플랫폼 엔지니어링

최종 카테고리 명칭은 구현 과정에서 조정할 수 있지만, 제품 계약은 처음부터 카테고리 기반 탐색을 전제로 해야 한다.

## 인증과 접근 제어

- 첫 릴리즈에서 제품은 공개 읽기 서비스가 아니다.
- 사용자는 큐레이션된 기사 상세 화면에 접근하기 전에 반드시 인증되어야 한다.
- 승인 소스 관리나 처리 실패 검토 같은 관리자 흐름은 일반 독자 경험과 분리해 후속으로 추가할 수 있다.
- 정확한 인증 제공자 선택은 구현 상세이지만, PRD는 인증 세션이 필요하다는 점을 전제로 한다.

## 비목표

- 첫 릴리즈에서 일반 소셜 네트워크나 토론 커뮤니티를 만들지 않는다
- 최대한 많은 콘텐츠 양을 목표로 하는 일반 크롤러를 만들지 않는다
- 임의 링크를 누구나 제출하는 오픈 플랫폼을 만들지 않는다
- 첫 릴리즈에서 완전한 노트 앱이나 지식 관리 도구를 만들지 않는다
- 첫 릴리즈에서 금융 지식 플랫폼을 만들지 않는다
- 장기적인 슈퍼 웹 비전을 한 번에 모두 구현하지 않는다

## 사용자 스토리

- 인증된 사용자로서, 유용한 기술 글을 한국어로 번역되고 설명된 형태로 한곳에서 읽고 싶다
- 개발자로서, 낯선 용어를 따로 검색하지 않도록 기사 단위 용어 설명과 개념 설명을 보고 싶다
- 학습자로서, 한 글만 읽고 끝나는 것이 아니라 관련 개념을 통해 학습을 이어가고 싶다
- 운영자나 제품 책임자로서, 승인된 소스와 카테고리가 일관되게 유지되어 지식 라이브러리의 신호 품질이 높게 유지되길 원한다

## 완료 기준

- 사용자는 주요 지식 경험에 접근하기 전에 반드시 인증해야 한다
- 사용자는 처리된 기사를 소스와 카테고리 기준으로 탐색할 수 있어야 한다
- 각 기사 상세 화면은 구조화된 한국어 번역, 요약, 용어 설명, 개념 설명, 관련 개념을 포함해야 한다
- 첫 릴리즈는 기술 콘텐츠에 한정되고, 금융 확장은 후속 단계로 남아 있어야 한다
- 인증형 웹 경험을 지원하는 API는 일관된 응답 및 오류 형태를 가져야 한다

## 성공 지표

- 첫 유의미한 이해까지 걸리는 시간: 로그인 후 짧은 시간 안에 하나의 유용한 기사를 읽고 이해를 시작할 수 있다
- 이해 품질: 사용자가 번역과 설명 덕분에 글을 이해하는 시간이 줄었다고 느낀다
- 지식 탐색 유용성: 사용자가 한 페이지에서 이탈하지 않고 카테고리와 관련 개념을 따라 계속 탐색한다
- 콘텐츠 품질: 승인된 소스에서 나온 구조화 산출물이 신뢰 가능하고 유용하다고 평가된다
- 확장 준비도: 첫 릴리즈는 작게 유지하지만 데이터 모델과 서비스 경계는 향후 소스 확장에 대응할 수 있다

## 미래 방향

장기적으로는 사용자가 여러 종류의 큐레이션된 정보를 한곳에서 접근할 수 있는 마이크로서비스 기반 지식 플랫폼을 지향한다. 다만 초기 릴리즈는 기술 콘텐츠 워크플로우를 먼저 검증해야 한다.

## 메모

이 PRD는 대규모 기능 확장이나 인프라 확장에 앞서 첫 릴리즈 제품 계약을 고정하기 위한 문서다. 소스 라이선스, 저장 정책, 번역 품질, 인증 경계는 구현 계획의 일부로 반드시 검토해야 한다.
