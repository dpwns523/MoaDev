# 제품 계획

## 목적

이 문서는 PRD를 실제 구현 기준의 MVP 계획으로 압축한 문서다. 장기 플랫폼 비전보다 훨씬 좁은 범위를 다루며, 첫 번째로 출시 가능한 제품에 집중한다.

기본 참조 문서:

- `docs/prd.md`
- `docs/prd.ko.md`

## 첫 릴리즈 정의

`MoaDev`의 첫 릴리즈는 기술 콘텐츠를 구조화된 한국어 형태로 읽을 수 있게 해주는 비공개 인증형 AI 지식 웹 애플리케이션이다.

로그인한 사용자는 다음을 할 수 있어야 한다.

- 승인된 소스에서 유용한 기술 글을 찾는다
- 처리된 기사 상세 화면을 연다
- 줄 단위 한국어 번역과 구조화된 설명을 읽는다
- 카테고리와 관련 개념을 통해 학습을 이어간다

## 주요 사용자

### 사용자 1: 실무 개발자

- 유용한 기술 콘텐츠를 빠르게 이해하고 싶다
- 여러 영어 원문 사이트를 매일 직접 뒤지고 싶지 않다
- 번역 정확도와 짧고 명확한 맥락 설명을 중요하게 생각한다

### 사용자 2: 기술 학습자

- 요약만이 아니라 개념 설명과 용어 설명도 원한다
- 한 글에서 끝나는 것이 아니라 인접 주제로 이어지는 학습 경로를 선호한다
- 카테고리 구조와 관련 개념 탐색을 중요하게 본다

### 사용자 3: 기술 리드

- 의미 있는 글을 빠르게 훑을 수 있는 고신호 읽기 화면을 원한다
- 신뢰 가능한 설명과 일관된 소스 선정이 중요하다

## MVP 사용자 여정

1. 사용자가 로그인한다.
2. 사용자는 카테고리와 소스 진입점을 가진 지식 홈에 도착한다.
3. 사용자는 카테고리 목록 또는 소스 목록을 연다.
4. 사용자는 기사 하나를 선택한다.
5. 사용자는 다음 구조화 섹션을 읽는다.
   - 기사 개요
   - 줄 단위 번역
   - 요약
   - 용어집
   - 개념 설명
   - 관련 개념
6. 사용자는 카테고리 또는 관련 개념 링크를 따라 탐색을 이어간다.

## MVP 화면

### 1. 로그인 게이트

- 인증 성공 전에는 모든 콘텐츠 접근을 막는다
- 로그인 후 원래 목적지로 돌아가는 세션 복구를 지원한다

### 2. 지식 홈

- 카테고리를 강조한다
- 최근 또는 추천 처리 완료 기사를 보여준다
- 사용자가 소스와 카테고리 기준으로 이동할 수 있게 한다

### 3. 카테고리 또는 소스 목록

- 처리된 기사 제목, 소스, 카테고리, 발행일, 처리 상태를 보여준다
- 첫 릴리즈에서는 필터를 최소화한다

### 4. 기사 상세

- 제품의 핵심 화면이다
- 기사 메타데이터, 소스 맥락, 구조화된 번역, 요약, 용어 설명, 개념 설명, 관련 개념을 보여준다
- AI 가공이 불완전하면 부분 완료 또는 열화 상태를 명확히 표시한다

## MVP 기능 우선순위

### 반드시 포함

- 인증 기반 접근
- 승인된 소스 레지스트리
- 하나의 소스 계열에 대한 기사 수집 및 AI 가공 파이프라인
- 소스 및 카테고리 탐색
- 구조화된 기사 상세 산출물
- 처리 상태 표시

### MVP 직후 빠르게 추가할 것

- 더 많은 IT 기업 기술 블로그 확장
- 더 나은 카테고리 큐레이션
- 간단한 읽기 이력 또는 이어보기 기능

### 보류

- 공개 접근
- 사용자가 임의로 제출하는 오픈 링크
- 메모, 북마크, 협업 기능
- 금융 지식 수집
- 비기술 분야까지 포괄하는 슈퍼앱 내비게이션

## 콘텐츠 거버넌스

- 첫 릴리즈에서는 승인된 소스만 허용한다.
- 소스별 라이선스와 저장 정책은 수집 활성화 전에 검토해야 한다.
- 제품은 구조화된 지식 경험을 위해 필요한 수준의 정규화 텍스트를 저장하되, 소스 정책 제약을 지켜야 한다.
- 신뢰도가 낮은 산출물은 상태 표시를 붙이거나 검토 전까지 공개하지 않아야 한다.

## 품질 기준

- 번역은 세그먼트 단위로 정렬되어 있어야 하고 읽기 쉬워야 한다
- 요약은 원문 의미를 왜곡하지 않아야 한다
- 용어 설명은 사전식 정의가 아니라 기사 맥락에 맞아야 한다
- 관련 개념은 임의의 키워드 확장이 아니라 학습에 실제 도움이 되어야 한다

## 릴리즈 순서

### 단계 0: 제품 계약 확정

- PRD, 에이전트 역할 계획, 프로덕션 계획 문서 확정

### 단계 1: 클로즈드 MVP

- 사용자 인증
- `Keek news` 수집
- 카테고리 기반 기사 목록과 기사 상세 화면 공개

### 단계 2: 소스 확장

- 선별된 IT 기업 기술 블로그 추가
- 카테고리 범위와 운영자 워크플로우 개선

## 열린 결정 사항

- 외부 소스별 온보딩 정책
- UI에 노출할 최종 카테고리 명칭
- 저장한 글 기능을 MVP에 넣을지, 직후 단계로 미룰지
- 최종 인증 제공자 선택
